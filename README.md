# etl-flow-module-03

## Структура решения
- `docker-compose.yml` - файл для развертывания Airflow, PostgreSQL, MongoDB и Streamlit.
- `Dockerfile` - образ Airflow с необходимыми зависимостями.
- `init.sql` - скрипт инициализации PostgreSQL.
- `streamlit_app.py` - дашборд для визуализации данных.
- `dags/`
  - `1_generate_data_mongodb.py` - DAG для генерации синтетических данных.
  - `2_mongodb_to_postgres_etl.py` - DAG для ETL процесса(`ods`).
  - `3_build_data_marts.py` - DAG для построения аналитических витрин(`dm`).

### 1. Процесс Репликации (DAG 2)
Осуществляет перенос данных из mongo в psql ods.
- **Экстракция**: Получение данных из MongoDB через `MongoHook`.
- **Трансформация**: Очистка от дубликатов, преобразование вложенных массивов и объектов в форматы `TEXT[]` и `JSONB` для корректного хранения в Postgres. Процесс исправляет типичные ошибки сериализации.
- **Загрузка**: Атомарная загрузка с использованием механизма `safe_load` (удаление существующих записей по ключам перед вставкой), что обеспечивает полную идемпотентность пайплайна.
- **Очистка в источнике**: После подтверждения успешной вставки в PostgreSQL, только загруженные записи удаляются из MongoDB.

### 2. Создание Витрин (DAG 3)
На основе чистых данных из схемы `ods` формируются аналитические витрины в схеме `dm`:
- `user_activity_mart`: анализ сессий, длительности и активности пользователей.
- `support_efficiency_mart`: статистика обращений в поддержку по типам и статусам, расчет времени решения.

### 3. Качество данных и Партиционирование
Хранящиеся данные чистые - не имеют дублей, корректно партиционированы по времени (range partitioning по годам), что делает их оптимальными для аналитики.

## Запуск
1. Запустите сборку и старт сервисов:
   ```bash
   docker-compose up -d --build
   ```
2. Airflow будет доступен по адресу `http://localhost:8080`
   - **Login:** admin
   - **Password:** admin
3. Зайдите в Airflow UI и запустите последовательно:
   - `1_generate_data_mongodb`
   - `2_mongodb_to_postgres_etl`
   - `3_build_data_marts`
4. **Дашборд Streamlit** доступен по адресу: `http://localhost:8501`
   - Там можно просмотреть содержимое таблиц и графики по витринам.
5. Параметры подключения к PostgreSQL:
   - Host: `localhost`, Port: `5432`
   - БД: `dwh_db`, User: `dwh_user`, Password: `dwh_password`
   - Витрины: `dm.user_activity_mart`, `dm.support_efficiency_mart`
6. Остановить контейнеры(удалить vol `-v`):
   ```bash
   docker-compose down
   ```
